# XGBoost 기반 Sysmon 로그 위협도 학습 파이프라인

이 문서는 EDR 프로젝트 내에서 시스템 로그(Sysmon)의 위험도를 판별하기 위해 구성된 **XGBoost 머신러닝 학습 환경**에 대한 진행 상황과 앞으로의 계획을 정리한 문서입니다.

작성일: 2026-05-22

## 1. 지금까지 완료된 작업 (Done)

- **폴더 및 환경 구성**
  - 모델 학습 전용 스크립트와 아티팩트를 관리하기 위한 `xgboost/` 폴더를 생성했습니다.
  - 모델 학습에 필요한 패키지(`xgboost`, `scikit-learn`, `joblib`, `pandas`)를 로컬 환경에 설치했습니다.
  - 최상위 폴더의 `requirements.txt`에 해당 패키지들을 추가하여 종속성 관리를 일원화했습니다.

- **범용 자동 학습 스크립트 작성 (`train.py`)**
  - **데이터 로드:** `.csv`, `.json`, `.xlsx` 포맷을 유연하게 로드할 수 있도록 구현했습니다.
  - **자동 전처리 (Auto Preprocessing):** 데이터셋에 문자열(카테고리) 데이터나 결측치가 포함되어 있어도 학습이 중단되지 않고 자동 처리되도록 구성했습니다.
    - 문자열 데이터: `LabelEncoder`를 사용하여 자동으로 수치형 데이터로 인코딩.
    - 결측치: 기본적으로 '0' 또는 'Missing'으로 처리.
  - **CLI 실행 지원:** 스크립트를 하드코딩하지 않고 터미널에서 데이터셋 경로와 타겟(정답) 컬럼을 지정하여 유연하게 사용할 수 있습니다.
  - **산출물 저장:** 학습이 완료된 XGBoost 모델 파일(`xgboost_sysmon_model.json`)뿐만 아니라, 추후 추론(Inference) 시 문자열 변환에 사용될 라벨 인코더 객체(`label_encoders.pkl`)도 함께 저장하도록 구현했습니다.

- **실제 데이터셋 기반 모델 학습 (2026-05-22 완료)**
  - `Dataset/normal` 및 `Dataset/malicious` 폴더의 JSON 데이터셋을 로드하고 하나로 병합하는 로직을 추가했습니다.
  - `LabelEncoder`를 거친 후 데이터를 강제로 정수형(`int`)으로 변환하여 XGBoost 데이터 타입 오류를 해결했습니다.
  - 정상 및 악성 로그를 8:2 비율로 분리하여 99.92%의 정확도로 학습을 완료했습니다.
  - 상세 내역을 `xgboost_training_log.md`에 문서화했습니다.

- **모델 성능 테스트 및 검증 (`test_model.py`)**
  - `Dataset/test` 폴더에 분리되어 있는 정상(`test_normal.json`) 및 악성(`test_malicious.json`) 데이터를 병합하고 셔플하여 테스트 스크립트를 구현했습니다.
  - 학습되지 않은 새로운 데이터(Unseen data)에 대한 안전한 인코더 변환(`safe_transform`) 로직을 적용하여 OOV(Out-Of-Vocabulary) 에러를 방지했습니다.
  - 테스트 세트 평가 결과 **약 94.65%의 정확도**를 검증했습니다.

- **Google Colab 전용 시각화 환경 구축 (`colab_visualization.ipynb`)**
  - 로컬 환경 외에도 클라우드(Colab) 상에서 모델의 성능을 다각도로 분석할 수 있는 주피터 노트북을 작성했습니다.
  - Train/Test 데이터 업로드, 모델 재학습, 그리고 평가 결과를 도출하는 엔드투엔드 파이프라인을 구성했습니다.
  - 분류 성능 리포트, 혼동 행렬(표 및 Heatmap), 클래스 분포(Pie 차트), 피처 중요도(Bar 차트), ROC 곡선, Precision-Recall 곡선, 예측 확률 분포(KDE) 등 풍부한 시각화 차트를 통합했습니다.
